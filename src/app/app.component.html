ANG-GCE-ROUTING

This demo will show Angular routing and route guards. Routing is a process by which single page applications show some content while hiding other content in order to emulate navigating between multiple HTML pages. Below are three routes, each leading to a city somewhere in the world. When we navigate to one of the routes we will see one of three components get rendered on screen.

Each route is also protected by a route guard, which allows us to control which routes are active. These routes are all inactive by default, indicated by the lock images. Click the locks to unlock (activate) that route. Then you will be able to click the "Navigate" button and navigate to that route.
